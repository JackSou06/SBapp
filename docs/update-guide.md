# Sparkle Update Guide

This project is an Electron macOS app. The packaged app includes Sparkle-compatible
metadata in `Info.plist`:

- `SUFeedURL`: `https://JackSou06.github.io/SBapp/appcast.xml`
- `SUPublicEDKey`: `WAaoS0fV2lGaAjhfbjYpeDmPxE7slzPC+AvJzz8j6X4=`

The repo is an Electron macOS app, so the update button calls a small native
Sparkle bridge instead of a SwiftUI `@main` app. The bridge lives in
`native/SparkleUpdateHelper` and uses Sparkle 2 `SPUUpdater`.

## Generate Sparkle Keys

Install or download Sparkle 2, then run:

```sh
generate_keys
```

Copy the generated public EdDSA key into `SUPublicEDKey` in `package.json`.
Keep the private key outside the repo and outside GitHub. Never commit it.

## Add Sparkle 2 for Release Builds

Download Sparkle 2 and place the framework at:

```text
native/vendor/Sparkle.framework
```

Alternatively, keep it outside the repo and set `SPARKLE_FRAMEWORK_PATH` when
running `npm run dist`.

Build the native helper:

```sh
scripts/build_sparkle_helper.sh native/vendor/Sparkle.framework
```

Or build the helper and package the macOS app in one step:

```sh
scripts/package_with_sparkle.sh native/vendor/Sparkle.framework
```

The helper is written in Swift and calls:

```swift
SPUUpdater(...).checkForUpdates()
```

During `npm run dist`, `scripts/after_pack_sparkle.cjs` copies the helper to
`Contents/Resources/SparkleUpdateHelper` and copies `Sparkle.framework` to
`Contents/Frameworks/Sparkle.framework` when those files exist.

## Package a New App

Build the app:

```sh
npm run dist
```

Create a zip archive for Sparkle:

```sh
scripts/release_update.sh "release/mac-arm64/Subtitle Combine.app" 1.2.3
```

Adjust the `.app` path to match the app generated by `electron-builder`.

## Upload to GitHub Releases

Create a GitHub Release for the new version and upload the generated zip file,
for example `Subtitle Combine-1.2.3.zip`.

## Generate `appcast.xml`

Use Sparkle's `generate_appcast` with the private key available locally:

```sh
generate_appcast /path/to/folder-containing-release-zips
```

Do not save the private key in this repo or in the script.

## Publish `appcast.xml`

Publish the generated `appcast.xml` to GitHub Pages so it is available at:

```text
https://JackSou06.github.io/SBapp/appcast.xml
```

The current `appcast.xml` in the repo is only a placeholder feed.

## Test an Update

1. Install an older packaged version into `/Applications`.
2. Publish a newer zip to GitHub Releases.
3. Generate and publish the new `appcast.xml` to GitHub Pages.
4. Launch the older app from `/Applications`.
5. Click `檢查更新` or use `Check for Updates...` from the macOS app menu.

The app can build before `appcast.xml` exists, but checking for updates will fail
until the feed is published and reachable.
